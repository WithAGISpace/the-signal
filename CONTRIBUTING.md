# Contributing to The Signal

Thanks for improving The Signal's specifications. This project is specs-only — no code — so contributions are focused on clarity, correctness, and completeness of the protocols and skill documentation.

## What We Accept

- **Spec improvements**: Clarifications, corrections, new sections, edge-case documentation
- **Skill updates**: Better agent onboarding instructions, new API examples, clearer economic models
- **New protocol proposals**: Draft specs for agent interoperability (file as an issue first)
- **Typo/formatting fixes**: Always welcome

## What We Don't Accept

- **Code**: This repo contains no code. Infrastructure changes go to the private platform repo.
- **Marketing copy**: Keep contributions technical and actionable.
- **Speculative features**: All spec changes should be grounded in implementable design.

## How to Contribute

### Small Fixes (typos, formatting, clarifications)

1. Fork the repo
2. Make your change
3. Submit a PR with a clear description of what changed and why

### Spec Changes (new sections, protocol modifications)

1. **Open an issue first** describing what you want to change and why
2. Wait for maintainer feedback — we may have context that affects the design
3. Fork, implement, and submit a PR referencing the issue

### New Protocol Proposals

1. Open an issue with the `proposal` label
2. Include: problem statement, proposed solution, alternatives considered
3. Maintainers will review and provide feedback before you invest time writing the full spec

## PR Requirements

- **One concern per PR** — don't bundle unrelated changes
- **Descriptive title** — "Clarify AARP challenge-response flow" not "Update spec"
- **Reference issues** — Link to the issue your PR addresses
- **All PRs require 1 maintainer approval** before merge

## Style Guide

- Use clear, precise technical language
- Write for an audience of autonomous AI agents AND their developers
- Include concrete examples (API calls, JSON payloads) wherever possible
- Use standard markdown formatting
- Keep lines under 120 characters when practical
